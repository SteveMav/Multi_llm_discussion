Étape 02 — Armement — Création de Session

Créez une nouvelle session de débat en définissant les paramètres fondamentaux. Un « sanity check » automatique valide la cohérence avant d'armer la session.

Paramètres Requis
_TITLE

Nom de la session (ex: « Alignment éthique des LLMs »)

_TOPIC

Le sujet de débat — c'est le prompt central qui sera soumis aux agents

_TOKEN_BUDGET

Budget maximum de tokens pour la session (100 – 1 000 000)

✓

Après validation, le statut passe à READY et vous êtes redirigé vers la Roundtable.

4
Étape 04 — Le Cockpit — Observation Live

Le cœur de l'observatoire. Le cockpit affiche le débat en temps réel via Server-Sent Events (SSE). Chaque agent est identifié par son waypoint chromatique.

Les 3 Phases du Débat
Exposition

Chaque agent présente sa position initiale sur le sujet

Confrontation

Rounds d'argumentation adversariale entre agents

Résolution

Le modérateur synthétise et produit le verdict final

💡 Waypoints Chromatiques

Chaque intervention d'un agent est colorée selon la couleur définie dans la Genetic Matrix. Identifiez instantanément qui parle sans lire le label.

5
Étape 05 — Contrôle & Export

Deux outils de clôture pour chaque session : l'arrêt d'urgence et l'exportation du rapport.

⚠ Kill-Switch

Si un débat dérape (boucle infinie, hallucinations, perte de cohérence), activez le Kill-Switch. Une justification est obligatoire pour l'audit trail.

session.status → ABORTED | abort_justification: string (required)
↓ Export PDF

Une fois le débat terminé (statut SUCCESS ou ABORTED), téléchargez le rapport forensique PDF. Il contient l'intégralité des échanges et métadonnées.

endpoint: /session/{id}/report/pdf/ | content-type: application/pdf
